MAP-EDITOR


ABOUT

The CAMPS database holds crucial objects that defined the paths drawn by CAMPS. Map-Editor is a GUI that allows an administrator to quickly and seamlessly change these database objects in a way intuitive to humans. All changes are updated real-time into the database.

The three database objects are:
Building Nodes
Represents a building on campus. Each node is defined in terms of <Name, Abbreviated name, Node ID, Pixel X-location, Pixel Y-location>.
    • Name – The full name of the building, as defined by http://www.washington.edu/home/maps. For example, Chemistry Library Bldg.
    • Abbreviated Name – The abbreviated name of the building (also defined by the above website). For example, CHL.
    • Node ID – The database integer id key particular to this node.
    • Pixel X-location – The x-value of the node location, with the origin centered at the top-left corner of the campus map.
    • Pixel Y-location – The y-value of the node location.
Path Nodes
Represents a non-building location on campus. They define the path between buildings. Each node is defined in terms of <Node ID, Pixel X-location, Pixel Y-location>. Node ID, Pixel X-location, and Pixel Y-location are the same as for Building Nodes. Note that Building Nodes and Path Nodes are actually both node types in the database.
Edges
Represent paths between two nodes on campus. These can be either Building Nodes or Path Nodes. Each edge is defined in terms of <Node1, Node2, Direction, Path Type>.
    • Node 1 – The start node defining the edge.
    • Node 2 – The destination node defining the edge.
    • Direction – Either bi-directional or not bi-directional. Bi-directional means the edge can be traveled from node1 to node2 or vice versa. Not bi-directional means the edge can only be traveled from node1 to node2. This may the be case for one-way doors.
    • Path Type – Currently, either Default or Stairs. Default represents all non-stairs paths. The Path Type is important in cases where traveling is done a bike, wheelchair, or other mechanism that might make stair travel difficult or impossible. Future extensions of Path Type and traveling method can result in more variations.

Map-Editor allows an admin to change these objects with powerful built-in GUI tools, layed out on the back-drop of the actual UW Campus, so WYSIWYG (what you see is what you get). Please read the User’s Manual to learn how to use Map-Editor.

Administrator Instructions:
Map-Editor is visually split up into 4 sections, from top to down.
    1. The Menu Toolbar (File, Change Colors).
    2. The Output Textbox Pane.
    3. The Map Interface.
    4. The Bottom Control Panel.
We will discuss each in detail, in the order that you should learn it. Map Interface, Bottom Control Panel, Output Textbox Pane, and Menu Toolbar.

Map Interface
- Right-click and drag to move around the map.
- Building nodes are represented by the sold blue circle with the building’s abbreviated name attached.
- Path nodes are represented by the blue concentric circle.
- Edges/paths are represented by the green lines. They are by default bidirectional, which is indicated by to arrows that point in opposite directions (looks like a bow-tie).

Bottom Control Panel
First, let’s discuss the radio buttons. They are Add Node, Add Building, Add/Edit Path, Remove Node, and Move Node.
Add Node:
Allows you to add Path Nodes. Click anywhere on the Map Interface to add a Path Node at that location. A blue concentric circle appears on the Map Interface, indicating the position of the Path Node.
Note: for all radio button options, both left-click and right-click work. But if you right-click and drag (to move around the map) the radio button action will not be taken.
Add Building:
Allows you to add Building Nodes.
    1. Click anywhere on the Map Interface to add a Building Node at that location.
    2. An input textbox pop-up will ask for the “Full Name of the new Building”. Type in the full-name of the building. Eg, Kane Hall.
    3. Press OK.
    4. An input textbox pop-up will ask for the “Abbreviation of the new Building”. Type in the abbreviation of the building. For example, KNE.
    5. Press OK.
    6. A solid blue circle appears on the Map Interface, indicating the position of the new Building Node.
Add/Edit Path:
Allows you to create/edit the edge/path between two nodes.
    1. Click on the first node (either Path Node or Building Node) defining the edge you want to create (or already exists). The node will be highlighted green, indicating your selection.
    2. Click on the second node defining the edge. The node will be highlighted red, indicating your selection. A pop-up menu will appear.
        • Checkmark the Bi-directional box to make the edge bi-directional (traveling is allowed either from 1st node to 2nd node or vice versa).
        • Uncheckmark the Bi-directional box to make the edge non-bi-directional (traveling is only allowed from the 1st node to the 2nd node).
        • Arrows along the path on the Map Interface visually show which directions are allowed.
        • Select the Path Type from the drop down menu. Currently, Default and Stairs are supported.
        • Click the Remove Edge button (only appears if the edge already exists) to delete the edge.
        • Click OK to finalize your selection.
        • Click Cancel to cancel your selection.
Remove Node:
Allows you to remove nodes (either Path Nodes or Building Nodes). Click on a node in the Map Interface, and a pop-up box will appear, asking “You have clicked on a node, would you like to delete it?” Click Yes to delete the node. Click No to not delete the node.
Move Node:
Allows you to move nodes. Click on a node in the Map Interface. The node will be highlighted green, indicating your selection. Use your keyboard arrow keys to move the node up, left, down, or right. This allows for small adjustments. If you want to move a node large distances, delete the node and a new one.

Now, let’s discuss the other functions on the Bottom Control Panel.
Path Defaults Drop-down Menu:
Allows you to change the default path type. Once changed, new edges created will have the path type defaulted to the new value.
Bi-Directional Checkbox:
Allows you to change the default directional mode. Check it to make new edges default to bi-directional. Uncheck it to make new edges default to uni-directional.
Refresh Data:
Allows you to refresh the Map Interface. If someone else is also using the Map-Editor concurrently, Refresh Data updates the changes they made to your Map Interface.
Reset:
Clears all selections on the Map Interface. For example, if you have a selected a node, clicking Reset will clear your selection.
Show Picture Checkbox:
Uncheck this to not display the campus map background. The nodes all the edges will still display. May improve scrolling performance.
Non-Building Destinations Checkbox:
Uncheck this to not display Path Nodes. Allows you to focus on the Building Nodes and the paths between them.

Output Textbox Pane
Displays information about your interaction with the Map Interface. When you add a Path Node or a Building Node, it will display “Added new node” or “Added new building”. When you create/edit a path, it will display node information in the form [Node Id] at [X-Pixel],[Y-Pixel].

Menu Toolbar
File – Allows you to exit the program.
Change Colors – Allows you to change the colors of Map Interface components (Node, Paths, Start Node for edges, End Node for edges).
